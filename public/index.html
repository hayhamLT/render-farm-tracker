Help & documentation

How the farm tracker discovers machines, keeps software current, and what to do when something looks off.

New machines join by themselves

A machine shows up within a few minutes of the tracker agent running on it. With a Deadline Worker that's automatic; without Deadline, install the agent once — via the commands below, or pushed to many machines at once with Group Policy / a startup script.

New Windows machine

Once it's on the dashboard, do this once, in person to allow silent installs (Windows can't enable this remotely):

  1. Open \\THIS-server\INSTALLERS
  2. Right-click Elevate-Tracker.cmd → Run as administrator
  3. Click Yes — done

The green shield appears within a minute. (Not on the dashboard yet? See Enroll a machine yourself below.)

New Mac

Nothing to do — with a Deadline Worker a Mac enrolls and elevates itself; the green shield appears on its own.

No Deadline, or it's stuck? Run the macOS commands below once.

How updates work

It spots what's old

Every machine reports what it has installed. Anything out of date lights up yellow.

Downloads once

The installer downloads to the server one time — every machine then grabs it over the local network.

Installs quietly

Updates install in the background with no pop-ups, and are double-checked afterward.

Hands-off option

Turn on Auto-deploy for an app in the Catalog and the server keeps it current everywhere by itself. Off unless you switch it on.

Graphics drivers

NVIDIA drivers are kept current automatically, just like apps. Each machine gets the newest driver its own card supports.

Right driver per card

Newer cards get the latest driver; older cards get the newest one that still works on them — shown as up to date, not behind.

Never during a render

A driver install briefly resets the card, so it waits until the machine is idle. A render is never interrupted.

Safe

If an install fails, the old working driver stays in place. Nothing is left broken.

Restart to finish

Most apply right away; if one needs it, use the ⏻ Restart button on the machine's card.

Machines without an NVIDIA card show n/a and are skipped.

Adobe & After Effects

Adobe apps update automatically through Adobe's own updater — quietly, with no downloads to manage, and never during a render.

Handled automatically

The tracker shows the latest After Effects version Adobe offers and rolls it out to any machine that's behind — no manual installer needed.

New versions can take a little while

A brand-new release may not appear right away — Adobe publishes it to its update feed on its own schedule. Once it does, the tracker picks it up and updates the fleet on its own.

Creative Cloud app

The Creative Cloud app updates itself — shown for info only.

What the badges mean

Online — the machine's OS icon (Windows/Apple) shows green when it has checked in recently
Offline — the OS icon turns red (machine off, or its agent stopped; it self-heals)
Ready — updates install silently, no prompts
Needs elevation — run Elevate-Tracker.cmd on it once (see above)
Up to date — newest known version is installed
Update available — a newer build of the same major; click to patch (safe, in-place)
New major available — a newer major version; opt-in side-by-side install
Not installed — product not found on the machine
Self-updating — Adobe keeps Creative Cloud current; not managed here
–n/a — the product doesn't apply to this machine (e.g. an NVIDIA driver on a Mac, or on a non-NVIDIA box)
Restart — on an online machine: reboots it (recovers a stuck agent or applies a driver); interrupts any active render
Wake — on an offline machine: powers it on via Wake-on-LAN

Troubleshooting

A machine shows offline but it's switched on

Give it ~90 seconds — that's the check-in grace. Still offline? Its agent stopped; it usually relaunches itself within a few minutes (built-in watchdog). If it doesn't come back, reboot the machine. (If it's actually powered off, use the ⏻ Wake button to turn it on.)

A machine is online but updates never run on it

The agent is stuck. It usually fixes itself; to fix it now, click the ⏻ Restart button on its card. (A restart interrupts any render on that machine.)

A machine never appears on the dashboard

It needs the tracker agent installed. With a Deadline Worker that's automatic; otherwise install it once — the commands above, or pushed via Group Policy / a startup script.

An update ran but the version didn't change

The tracker double-checks every install, so it flags one that didn't actually change. Usually it's Redshift bundled inside Cinema 4D — update Cinema 4D instead.

An install is stuck or taking too long

In Updates, press ✕ on its progress bar to stop it. Stuck jobs also clear themselves after 45 minutes.
