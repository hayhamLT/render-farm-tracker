New machines join by themselves

Any machine running a Deadline Worker is discovered and added to the dashboard automatically within a few minutes of coming online — nothing to install by hand.

New Windows machine

It appears on the dashboard by itself. One click unlocks silent installs (Windows requires this once, in person — it can't be done remotely):

  1. On that machine, open \\THIS-server\INSTALLERS
  2. Right-click Elevate-Tracker.cmd → Run as administrator
  3. Click Yes on the prompt — done

The green shield shows up on its card within a minute.

New Mac

Fully automatic — discovery and elevation. The Mac is enrolled, switched to a root agent, and shows the green shield with no action from you.

If one ever gets stuck, the manual commands below fix it.

How updates work

Track

Every machine reports its installed versions once a minute. The latest releases are checked online automatically, so outdated installs light up yellow.

Download once

An installer is fetched a single time to the server, then every machine pulls it over the LAN — fast, and nothing hammers the internet.

Install safely

Files are integrity-checked (SHA-256), installs run silently with no pop-ups, and the version is verified afterwards so a failed install can't pretend.

Queue & control

Updates roll out a few machines at a time; the rest queue. Any running install can be stopped from its progress chip.

Auto-deploy (optional)

Flip an app's Auto-deploy toggle in the Catalog and the server keeps it current across the whole fleet by itself — installs it where missing, updates where behind, one canary machine first then the rest. Off by default.

Catalog & settings

The Catalog lists every tracked app and its latest version (auto-detected per vendor). Each app's Track toggle adds or removes it from the dashboard, counts and updates. Set the installer download folder there too — keep it on your server/share, not a synced folder like Dropbox, so multi-GB installers don't sync.

What the badges mean

Online — the machine's OS icon (Windows/Apple) shows green when it has checked in recently
Offline — the OS icon turns red (machine off, or its agent stopped; it self-heals)
Ready — updates install silently, no prompts
Needs elevation — run Elevate-Tracker.cmd on it once (see above)
Up to date — newest known version is installed
Update available — a newer build of the same major; click to patch (safe, in-place)
New major available — a newer major version; opt-in side-by-side install
Not installed — product not found on the machine
Self-updating — Adobe keeps Creative Cloud current; not managed here

Troubleshooting

A machine shows offline but it's switched on

Give it two minutes — agents check in once a minute. Still offline? Its agent stopped — click the ↻ reboot button on the machine's card to restart it remotely (a reboot relaunches the agent on its own). On Windows, re-running Elevate-Tracker.cmd also works — both are safe to repeat.

A machine is online but updates never run on it

Its agent is alive but stuck — it still checks in, but queued installs sit waiting. The agent now self-restarts if this happens (a built-in watchdog catches a stalled agent on its own). To fix it instantly, click the ↻ reboot button on its card — that bounces the machine through Deadline even when the agent is hung, and a fresh agent picks up the queue. (A reboot interrupts any render running on that node.)

A machine never appears on the dashboard

Auto-discovery rides on Deadline — make sure the Deadline Worker is installed and running on it. No Deadline? Use the manual enrolment commands above.

An update ran but the version didn't change

The job is marked failed on purpose — the install is verified against the actual version on disk. Common cause: Redshift bundled with Cinema 4D won't downgrade or override; update Cinema 4D instead.

An install is stuck or taking too long

Open Updates and press the ✕ on its progress chip to stop it. Stale jobs are also cleaned up automatically after 45 minutes.
